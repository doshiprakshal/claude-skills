# Platform Engineering Skills

Planned: 15 skills. 15 built so far.

Each skill lives in its own subfolder under [`skills/`](./skills) with a `SKILL.md` (instructions Claude follows when the skill runs) and a `README.md` (what it does, how to invoke it, and a worked example).

## Built

| Skill | What it does |
|---|---|
| [`platform-review`](./skills/platform-review) | Top-level, holistic capability/adoption/consistency triage. |
| [`internal-developer-platform`](./skills/internal-developer-platform) | IDP abstraction-layer architecture and build-vs-buy decisions. |
| [`backstage-review`](./skills/backstage-review) | Backstage catalog consistency, plugins, TechDocs, templates. |
| [`golden-path-review`](./skills/golden-path-review) | Whether a paved-road workflow covers real use cases and stays current. |
| [`service-catalog-review`](./skills/service-catalog-review) | Catalog data model, coverage, and downstream trust — tool-agnostic. |
| [`multi-cluster-review`](./skills/multi-cluster-review) | Fleet topology, config consistency, placement, and failover. |
| [`self-service-review`](./skills/self-service-review) | What's self-service vs. manual, and guardrail appropriateness. |
| [`developer-experience-audit`](./skills/developer-experience-audit) | Data-driven friction mapping across the developer journey. |
| [`platform-maturity-assessment`](./skills/platform-maturity-assessment) | Stage-model scoring across capability dimensions. |
| [`template-review`](./skills/template-review) | Technical quality of a specific scaffolding/software template. |
| [`platform-cost-review`](./skills/platform-cost-review) | Cost allocation, shared-resource attribution, and accountability. |
| [`platform-security`](./skills/platform-security) | Whether secure-by-default is actually the path of least resistance. |
| [`platform-scaling`](./skills/platform-scaling) | Platform-team/tooling scaling bottlenecks ahead of org growth. |
| [`platform-roadmap`](./skills/platform-roadmap) | Sequences platform gaps into an impact/effort-prioritized roadmap. |
| [`platform-documentation`](./skills/platform-documentation) | Doc coverage, findability, and currency vs. real support questions. |
